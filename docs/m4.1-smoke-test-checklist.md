# M4.1 Smoke Test — reverse shadow-read 驗證（約 5 分鐘）

> **使用時機**：M4.1 合入 main 後，由 Tedus 在 GitHub 網頁操作，驗證：
> (1) reverse shadow-read consistent 路徑（開既有 feedback，List 有 → log consistent）
> (2) list_missing 路徑（List 端找不到該 id → log list_missing）

---

## Step 1：暫時切到 List primary

1. 開啟 `https://github.com/Tedus-AI/AI-Thermal-pad-and-stud-size-Evaluation-Tool/blob/main/config.js`
2. 按鉛筆圖示（Edit this file）
3. 將 `PRIMARY_FEEDBACK` 改為 `'list'`：
   ```js
   PRIMARY_FEEDBACK: 'list',
   ```
4. Commit message：`temporary: smoke test M4.1 reverse shadow-read`
5. 選 **Commit directly to `main`** → 按 Commit changes
6. - [ ] 開 Actions tab，等「pages build and deployment」出現綠勾 ✅（約 1–2 分鐘）
7. - [ ] Ctrl+F5 強制重整
8. - [ ] F12 → Console 確認：`window.FEATURE_FLAGS.PRIMARY_FEEDBACK === 'list'`

---

## Step 2：確認 SHADOW_READ_FEEDBACK 已開啟

- [ ] Console 確認：`window.FEATURE_FLAGS.SHADOW_READ_FEEDBACK === true`

  若為 `false`，回 config.js 將 `SHADOW_READ_FEEDBACK` 改為 `true`，
  一併 commit（commit message 後加 ` + enable shadow-read`），再等部署 + Ctrl+F5。

---

## Step 3：觸發 reverse shadow-read（consistent 路徑）

- [ ] 切到 Tab 5（使用者建議與改善）
- [ ] 點任意一筆的「閱讀 👁️」開啟 detail modal
- [ ] 關閉 modal

---

## Step 4：查看 shadow-read 報告（consistent 路徑驗證）

- [ ] 展開「🔧 開發者驗證（dual-write 測試）」→ 按「📊 查看 shadow-read 報告」
- [ ] **預期統計區**：
  - `consistent: 1`（或 `lossy_only: 1`）
  - `real_diff: 0`、`json_missing: 0`、`list_missing: 0`、`error: 0`
- [ ] Console 確認最新 entry 的 primary 欄位：
  ```js
  dbAdapter.getShadowReadLog()[0].primary  // 預期：'list'
  ```
- [ ] **若出現 `real_diff > 0`**：展開紅色 row 查看 diff，**先不要執行 Revert**，截圖並回報

---

## Step 4.5：驗證 list_missing 路徑（M4.1 必修 1）

目的：確認「List 端找不到該 id」時，log 有正確記錄 `list_missing`。

- [ ] F12 → Console，輸入並按 Enter：
  ```js
  dbAdapter.getDoc('feedback_items', 'FB-NONEXISTENT-TEST-' + Date.now())
  ```
- [ ] 預期 Console 顯示 Promise resolve → `null`
- [ ] 再按「📊 查看 shadow-read 報告」（或按開發者驗證區按鈕刷新）
- [ ] **預期**：最新一筆 entry（表格第一行）：
  - `result` 欄：`list_missing`
  - 摘要欄：`List 端找不到該筆`
  - 背景色：橙色（`#fff3e0`）
- [ ] Console 確認：
  ```js
  dbAdapter.getShadowReadLog()[0]
  // 預期 { id: 'FB-NONEXISTENT-TEST-...', result: 'list_missing', diffs: [], primary: 'list', ts: ... }
  ```
- [ ] 統計區 `list_missing: 1`（比剛才多 1）

---

## Step 5：Revert

1. 開啟 config.js（同 Step 1 路徑）
2. 按鉛筆圖示，將 `PRIMARY_FEEDBACK` 改回 `'json'`：
   ```js
   PRIMARY_FEEDBACK: 'json',
   ```
3. Commit message：`revert: M4.1 smoke test complete, restore PRIMARY_FEEDBACK='json'`
4. 選 **Commit directly to `main`** → 按 Commit changes
5. - [ ] 等 Actions tab 綠勾 ✅ → Ctrl+F5
6. - [ ] Console 確認：`window.FEATURE_FLAGS.PRIMARY_FEEDBACK === 'json'`
7. - [ ] 開幾筆 feedback detail，確認列表載入正常、既有 Phase 3 shadow-read 仍運作

---

|---|---|---|

| list_missing（Step 4.5） | result=list_missing | primary='list'，背景橙色，摘要正確 |
